INSTALLATION OF FLASK FRAMEWORK (macOS):

  1. Create and activate a virutal environment using python3 -m venv venv and source venv/bin/activate.
  2. Install python from python.org and verify using python3 --version.
  3. Install pip using pip installation guide and verify using pip --version.
  4. Make a directory for the flask project mkdir problem3-flask.
  5. Move into the directory with cd problem3-flask.
  6. Install flask using pip install flask.
  7. Create the folder/file structure and code (see in "DEVELOPMENT PROCESS OF WEBSITE USING FLASK FRAMEWORK" for details).
  8. Run program using flask run for testing.
  9. Install gunicorn python library using pip install flask gunicorn. This will be used to help run the Flask project when deployed.
  10. Create a requirements.txt file that will save the dependencies using pip freeze > requirements.txt.

DEVELOPMENT PROCESS OF WEBSITE USING FLASK FRAMEWORK:

  1. After installing all requirements, I developed the folder/file structure of my website:
    • Created a templates folder for my HTML files: base.html & development.html.
    • Created a static folder for CSS stylesheet and assets: styles.css.
    • Created an app.py used for the Flask framework backend.
    
                        
                        problem3-flask/
                        |── app.py              
                        |── templates/           
                        |   └── base.html    
                        |   └── development.html          
                        |   ── static/              
                            └── styles.css
  2. Set up the Flask backend in app.py:
    • Imported Flask and render_template.
    • Defined the routes for the HOME and DEVELOPMENT pages. This tells flask which URL to access each page with!
    • Rendered HTML templates using render_template().
    

                        from flask import Flask, render_template

                        app = Flask(__name__)

                        @app.route('/')
                        def home():
                            return render_template('base.html')

                        @app.route('/development')
                        def development():
                            return render_template('development.html')

                        if __name__ == '__main__':
                            app.run()
  3. Designed the HOME page in base.html:
    • Added a header (including the nav bar) and footer using HTML.
    • Defined { % block title % } { % endblock % } and { % block content % } { % endblock % } code blocks. These blocks will allow for page specfic content when we extend this file.
    • Added additional html code to create a landing page. For example, I added code to create the title.
    

                        For example, by placing { % block content % }{ % endblock % } code 
                        into the base.html file, any other file that extends base.html can 
                        place its unique html code within these braces. This will allow the
                        additional code to be rendered with the base.html header and footer!
  4. Designed the DEVELOPMENT page and added CSS:
    • Extended the base.html template to get the header and footer for the development.html page.

  5. Deployed the webpage:

CHALLENGES IN WEBSITE DEVELOPMENT:

  • Template Errors: Missing template files caused rendering errors when trying to create new routes. To solve this issue, I ensured all files were in the templates folder and routed correctly (e.g. proper spelling).
  • CSS Stylings: Some of the styles I created didn't apply as I intended them to. To solve this issue, I debugged removed and added styles, noticing I had conflicting style rules. After this discovery, I resolved the styling issue.
  • Folder Styling: In one portion of this project I wanted to display a folder/file structure, which would be challenging to do. To solve this issue I used the preformatted text tag to perserve whitespaces.
  • Hosting Difficulties: Deploying the website a was something that was new to me, timeconsuming, and challenging. To solve this issue, I watched youtube videos to understand how to host the webpage and was able to find a hosting site (Render). Here I needed to further learn about the gunicorn python library and creating a requirements.txt file, which allowed me to deploy the site.
